The component `<Suspense>` allows you to orchestrate the loading of resources
inside (nested) child components. Suspense is a light abstraction on top of a
[`<svelte:boundary>`](https://svelte.dev/docs/svelte/svelte-boundary), and will behave like one
with a few extra features.

<Example path="extras/suspense" />

## Usage

### In a child component

Let's have a look at a simple component that loads a model with the hook `useGltf`.

```svelte title="Model.svelte"
<script>
  import { T } from '@threlte/core'
  import { useGltf } from '@threlte/extras'

  const gltf = await useGltf('model.gltf')
</script>

<T is={gltf.scene}>
```

This component is already _suspense-ready_ by using top level await. If it is a child of `<Suspense>`,
it will not show until the gltf has loaded.

<Tip type="tip" title="Suspense-ready">
  *Suspense-ready* means it has the ability to hit a `<Suspense>` boundary if there is any. If there's no `<Suspense>` boundary, the component will behave as usual.
</Tip>

### In a parent component

Now we implement the component "Model.svelte" in a parent component:

```svelte title="Parent.svelte"
<script>
  import Model from './Model.svelte'
</script>

<Model />
```

To let the parent component decide what to do while the model component is loading, we wrap the child component in a `<Suspense>` component and show a pending component while the child component is loading.

```svelte {3,4,7,8,10}+ title="Parent.svelte"
<script>
  import Model from './Model.svelte'
  import Pending from './Pending.svelte'
  import { Suspense } from '@threlte/extras'
</script>

<Suspense>
  <Model />

  {#snippet pending()}
    <Pending />
  {/snippet}
</Suspense>
```

<Tip type="tip" title="Error Boundary">
  The `<Suspense>` component also acts as an error boundary for async requests made in child components wrapped in `suspend`.

The `"failed"` snippet will be mounted as soon as an error is thrown in a child component wrapped in `suspend` and the snippet prop `error` can be used to display a meaningful error message.

</Tip>

## UX considerations

The `<Suspense>` component is a great tool to improve the user experience of your application. However, it is important to consider the following points:

1. The `<Suspense>` component will only ever show one of the available slots at any time:

- The snippet `"failed"` will be shown as soon as an error is thrown.
- The snippet `"pending"` will be shown as long as a child component is suspended.
- The default slot will be shown as long as no child component is suspended and no error has been thrown.

2. Umounting a component that suspends rendering through awaiting a resource or throwing an error will discard its suspend state.
3. Don't put a Suspense boundary around every component. Suspense boundaries should not be more granular than the loading sequence that you want the user to experience.

## Limitations

1. It's important to keep in mind that while the contents of the default slot are not visible, they
   are still mounted and the component is otherwise fully functional. This means that any side
   effects (for instance `useTask` hooks) that are invoked in a _suspense-ready_ child component will be
   executed immediately.

```svelte title="Model.svelte"
<script>
  import { T, useTask } from '@threlte/core'
  import { useGltf } from '@threlte/extras'

  useTask(() => {
    // This will be executed immediately
  })

  const gltf = await useGltf('model.gltf')
</script>

<T is={scene}>
```

2. Also, in contrast to the React Suspense API, the `<Suspense>` component does not support showing a stale version of a child component while it is loading.
