This component can be used to transform objects in 3D space by adapting a similar interaction model of DCC tools like Blender. Unlike other controls, it is not intended to transform the scenes camera.

The most common way to use `<TransformControls>` is to set it as the parent of the component to be transformed.

  iframe
/>

### Examples

```svelte title="Scene.svelte"
<script>
  import { T } from '@threlte/core'
  import { TransformControls } from '@threlte/extras'
</script>

<TransformControls>
  <T.Mesh>
    <T.BoxGeometry />
    <T.MeshStandardMaterial />
  </T.Mesh>
</TransformControls>
```

The `<TransformControls>` component can also be used to transform an object passed to it:

```svelte title="Scene.svelte"
<script>
  import { T } from '@threlte/core'
  import { TransformControls } from '@threlte/extras'
</script>

<T.Mesh>
  {#snippet children({ ref })}
    <T.BoxGeometry />
    <T.MeshStandardMaterial />

    <TransformControls object={ref} />
  {/snippet}
</T.Mesh>

<TransformControls object={someObject} />
```

To accommodate using `<TransformControls>` with a `<OrbitControls>`, `<CameraControls>`, or `<TrackballControls>` component in the same scene, `<TransformControls>` will automatically pause the currently active camera controlling component when the user is interacting with the `<TransformControls>`. You can opt out of this behaviour by setting the property `autoPauseCameraControls` to `false`.

You can also pass a class instance of a camera controller to disable that when interacting with the transform controls.

```svelte title="Scene.svelte"
<script>
  import { T } from '@threlte/core'
  import { TransformControls } from '@threlte/extras'

  let { myOrbitControls } = $props()
</script>

<TransformControls cameraControls={myOrbitControls}>
  <T.Mesh>
    <T.BoxGeometry />
    <T.MeshStandardMaterial />
  </T.Mesh>
</TransformControls>
```
