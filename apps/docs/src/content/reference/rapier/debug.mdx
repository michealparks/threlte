Use the `<Debug>` component to see live wireframe representations of all colliders in a scene. This is useful for visually verifying that your physics colliders match the shapes and positions of your 3D objects.

<Example path="rapier/debug" />

### Basic Usage

Add `<Debug />` anywhere inside a `<World>` to render all colliders:

```svelte
<World>
  <Debug />

  <!-- all colliders in the world will be visualized -->
</World>
```

### Debuggable Objects

The `<Debug>` component visualizes all collider shapes present in the physics world:

- **Primitive shapes** — `cuboid`, `ball`, `capsule`, `cylinder`, `cone`
- **Rounded variants** — `roundCuboid`, `roundCylinder`, `roundCone`
- **Mesh-based shapes** — `trimesh`, `convexHull`, `convexMesh`
- **Other shapes** — `segment`, `triangle`, `heightfield`, `polyline`
- **AutoColliders** — any shape generated by `<AutoColliders>`
- **Sensors** — sensor colliders are also rendered

### Styling

Since `<Debug>` extends `LineBasicMaterial`, you can pass any material properties to customize the wireframe appearance:

```svelte
<Debug
  color="red"
  depthTest={false}
  depthWrite={false}
  opacity={0.5}
  transparent
/>
```
